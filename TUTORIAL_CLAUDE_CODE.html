⚙️ 2. Instalacja Claude Code

Claude Code instaluje się globalnie przez npm (masz już Node.js).

1

Otwórz terminal w VSC

W VSC: menu Terminal → New Terminal lub skrót Ctrl+` (backtick). Otworzy się terminal zintegrowany z VSC.

2

Zainstaluj Claude Code globalnie

npm install -g @anthropic-ai/claude-code

Instalacja trwa 1–2 minuty. Po zakończeniu powinno pojawić się potwierdzenie.

3

Sprawdź instalację

claude --version

Powinno wyświetlić numer wersji np. 1.x.x.

4

Zaloguj się do Anthropic (pierwsze uruchomienie)

claude

Przy pierwszym uruchomieniu Claude Code otworzy przeglądarkę i poprosi o zalogowanie się do konta Anthropic (tego samego co claude.ai). Po autoryzacji wróć do terminala — gotowe.

✓ Weryfikacja kompletna gdy: Wpisanie claude w terminalu otwiera interaktywną sesję z promptem >. Wyjście: Ctrl+C lub wpisz /exit.
Błąd "permission denied" przy npm install -g (macOS/Linux): Nie używaj sudo. Zamiast tego skonfiguruj npm prefix:
mkdir ~/.npm-global && npm config set prefix '~/.npm-global'
Następnie dodaj do ~/.zshrc lub ~/.bashrc:
export PATH=~/.npm-global/bin:$PATH
Następnie: source ~/.zshrc i ponów instalację.

🖥️ 4. Claude Code w VSC — Terminal jako główne środowisko

Claude Code działa w terminalu VSC — masz podgląd kodu obok i terminal poniżej jednocześnie. To jest Twoje główne środowisko pracy.

1

Otwórz folder projektu w VSC

File → Open Folder → wybierz triathlon-planner/
Lub z terminala: code ~/Documents/triathlon-planner

2

Otwórz terminal zintegrowany

Ctrl+` (backtick) — terminal otwiera się automatycznie w folderze projektu.

3

Uruchom Claude Code

claude

Claude Code widzi automatycznie wszystkie pliki w bieżącym folderze. Wczyta CLAUDE.md jako kontekst.

4

Rozmawiasz normalnym językiem

> dodaj obsługę brick workouts do season_plan.py
... Claude analizuje plik, proponuje zmiany, pyta o potwierdzenie ...
> tak
... Claude edytuje plik bezpośrednio, możesz zobaczyć zmiany w VSC ...
Układ okna który sprawdza się najlepiej: W VSC: przeciągnij terminal na prawą stronę (View → Editor Layout → Two Columns). Kod po lewej, terminal z Claude Code po prawej — widzisz zmiany w czasie rzeczywistym.
📋 5. CLAUDE.md — serce projektu

CLAUDE.md to plik który Claude Code wczytuje automatycznie przy każdym uruchomieniu. Zastępuje całą historię rozmów z claude.ai — zawiera wszystko co Claude musi wiedzieć o projekcie.

Poniżej gotowy CLAUDE.md dla tego projektu — skopiuj do folderu:

# CLAUDE.md — Triathlon Training Planner

## Projekt
Generator planów treningowych dla triathlonistów.
Wgrywa treningi do Garmin Connect przez oficjalne API.
Dystanse: 70.3, full, olympic, sprint.

## Stack
- Python 3.10+
- garminconnect 0.3.x (pip)
- venv w folderze: garmin-venv/
- Aktywacja: source garmin-venv/bin/activate (Mac/Linux)

## Pliki kluczowe
- season_plan.py     — główny: plan sezonu, wiele zawodów
- generate_plan.py   — plan jednych zawodów
- season_example.json — szablon konfiguracji
- CHANGELOG.md       — historia zmian (aktualizuj przy każdej zmianie)

## Garmin API — potwierdzone empirycznie

### Power target (rower) — KRYTYCZNE
- interval/recovery: workoutTargetTypeId=2, workoutTargetTypeKey="power.zone"
  targetValueOne/Two = absolutne waty (np. 158.0, 184.0)
- warmup/cooldown: workoutTargetTypeId=1, workoutTargetTypeKey="no.target"
  targetValueOne/Two = null
- BŁĄD: targetTypeId=5 lub power target na warmup → Garmin wyświetla jako kph!
  (wartość_w_watach × 3.6 = wyświetlane_kph — odkryte przez analizę sieciową)

### Pace target (bieg)
- workoutTargetTypeId=6, workoutTargetTypeKey="pace.zone"
- wartości w m/s: 1000 / (minuty*60 + sekundy)

### Sport IDs
- running=1, cycling=2, swimming=4

### garminconnect 0.3.x
- Używa self.client zamiast self.garth
- Wykrywanie: getattr(client, "garth", None) or getattr(client, "client", None)
- DELETE: http.request("DELETE", "connectapi", "/workout-service/...", api=True)
- Kalendarz: GET /calendar-service/year/{year}/month/{0-indexed}

## Konwencje kodu
- Każdy wyścig ma unikalny PREFIX (np. WARSAW, BERLIN)
- --reset usuwa tylko treningi z danym prefixem (nie inne plany)
- Warmup/cooldown zawsze no.target — NIGDY nie dawaj im power/pace target
- Pliki .zwo generowane oddzielnie (nie ma ich jeszcze w tym repo)

## Wersjonowanie
- Aktualizuj CHANGELOG.md przy każdej zmianie
- Format commitów: feat:, fix:, docs:, refactor:
- Przykład: git commit -m "fix: warmup cooldown no.target for all bike steps"

## Planowane funkcje (backlog)
- Zapis tokenu sesji Garmin (brak potrzeby logowania przy każdym uruchomieniu)
- Repeat groups (interwały z powtórzeniami w natywnym formacie Garmin)
- Generator .zwo zintegrowany z pakietem
- Eksport CSV/PDF
- Automatyczne FTP z API Garmin/Strava
Zapisz jako CLAUDE.md w folderze projektu. Claude Code wczytuje go automatycznie. Nie musisz go wklejać — wystarczy że istnieje w folderze.
🚀 6. Pierwsza sesja — migracja z claude.ai

Jak wyglądają pierwsze minuty w Claude Code po migracji z tej rozmowy:

1

Uruchom Claude Code w folderze projektu

cd ~/Documents/triathlon-planner
source garmin-venv/bin/activate
claude
2

Claude automatycznie wczytuje CLAUDE.md — powiedz mu co chcesz

> Podsumuj co jest w tym projekcie i jaki jest stan na dziś

Claude przeczyta CLAUDE.md, CHANGELOG.md i pliki .py — odpowie z pełnym kontekstem bez żadnego wklejania.

3

Przetestuj że skrypty działają

> Uruchom generate_plan.py w trybie dry-run żeby sprawdzić czy działa

Claude wykona komendę i pokaże wynik bezpośrednio w sesji.

4

Dodaj pierwszą nową funkcję

> Dodaj do season_plan.py możliwość eksportu planu do pliku CSV
   (data, typ sportu, nazwa treningu, czas)

Claude pokaże diff, zapyta o potwierdzenie, doda funkcję i zaktualizuje CHANGELOG.md.

5

Commit przez Git Extension w VSC

W VSC kliknij ikonę Source Control (lewy pasek, ikona gałęzi). Zobaczysz zmienione pliki. Wpisz wiadomość commita i kliknij ✓.

Lub z terminala Claude Code:

> zrób commit z sensowną wiadomością
🔄 7. Codzienny workflow

Scenariusz A — Rozwijanie skryptów (Claude Code)

Otwórz VSC
Terminal → claude
Opisz co chcesz
Claude edytuje pliki
Git commit
# Przykłady poleceń do Claude Code
> dodaj obsługę dystansu "swim-run" do PROFILES
> napraw błąd który pojawia się gdy uruchomię --reset dla nieistniejącego prefixu
> refaktoruj funkcję clean_prefix żeby była szybsza (mniej requestów API)
> zaktualizuj CHANGELOG.md o dzisiejsze zmiany

Scenariusz B — Diagnostyka Garmin (claude.ai)

Screenshot problemu
Wklej na claude.ai
Claude diagnozuje
Pobierz poprawkę
Wklej do VSC

Nadal używaj claude.ai gdy masz screenshot z Garmin Connect lub potrzebujesz Strava MCP do analizy aktywności.

Scenariusz C — Nowy sezon / nowe zawody

# Edytuj plik konfiguracyjny
> dodaj do season_example.json nowe zawody BERLIN 2027-09-15 70.3

# Testuj
python season_plan.py --config season_example.json --dry-run

# Upload
python season_plan.py --config season_example.json --reset
Złota zasada workflow: Zawsze kończ sesję z Claude Code commitem. Nawet małe zmiany — git commit -m "docs: update CHANGELOG". Historia gita zastępuje historię czatu.

GitHub — opcjonalny ale zalecany następny krok

Gdy projekt działa lokalnie, możesz dodać zdalne repozytorium:

# Utwórz repo na github.com (przycisk "New repository")
# Następnie w terminalu:
git remote add origin https://github.com/TWOJ_LOGIN/triathlon-planner.git
git branch -M main
git push -u origin main

Od tego momentu VSC Git Extension synchronizuje zmiany przez przycisk ↑ w pasku statusu.
