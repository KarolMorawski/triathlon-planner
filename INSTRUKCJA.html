🐍 1. Instalacja Python

Python jest wymagany do uruchomienia skryptów. Wersja minimalna: 3.10.

Windows — najczęstszy błąd: brak "Add to PATH" Pominięcie tego checkboxa powoduje błąd "python is not recognized". Postępuj dokładnie poniżej.
1

Pobierz instalator

Wejdź na python.org/downloads → kliknij żółty przycisk "Download Python 3.x.x" (wybierz 3.11 lub nowszy).

2

ZAZNACZ "Add Python to PATH" przed instalacją

W oknie instalatora, zanim klikniesz Install Now, zaznacz checkbox na dole: "Add Python 3.x to PATH". Bez tego Python nie będzie dostępny w terminalu.

3

Otwórz terminal i sprawdź instalację

Naciśnij Win + R, wpisz cmd, Enter. Następnie:

python --version   # powinno pokazać Python 3.11.x
pip --version      # powinno pokazać pip 23.x
Jeśli "python" nadal nie działa po instalacji: Zamknij i otwórz terminal ponownie (Path zmienia się dopiero po restarcie). Jeśli nadal błąd — zainstaluj Python ze sklepu Microsoft Store: wciśnij Win, wpisz "Python", kliknij "Python 3.11" w wynikach Store.
Windows — problem z PowerShell "execution of scripts is disabled" Uruchom PowerShell jako administrator i wpisz:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
1

Sprawdź czy Python jest już zainstalowany

Otwórz Terminal (Cmd+Spacja → "Terminal") i wpisz:

python3 --version

Jeśli wyświetla Python 3.10+ — przejdź od razu do sekcji 3. Jeśli nie — zainstaluj przez Homebrew.

2

Zainstaluj Homebrew (menadżer pakietów dla macOS)

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Wklej do Terminala, naciśnij Enter. Proces trwa kilka minut i może poprosić o hasło administratora.

3

Zainstaluj Python

python3 --version   # weryfikacja

⌚ 5. Logowanie do Garmin przez terminal

Skrypty Python łączą się z Garmin Connect przez oficjalne API. Przy pierwszym uruchomieniu skrypt zapyta o login i hasło. Dane nie są nigdzie zapisywane — podajesz je jednorazowo przy każdym uruchomieniu.

Co skrypt robi przy logowaniu

1

Pyta o email i hasło

Hasło jest wpisywane w trybie ukrytym — znaki nie pojawiają się na ekranie. To standardowe zachowanie dla bezpiecznego wpisywania haseł w terminalu.

2

Obsługuje weryfikację dwuetapową (2FA/MFA)

Jeśli masz włączoną weryfikację dwuetapową w Garmin Connect, skrypt automatycznie poprosi o kod po wpisaniu hasła. Wpisz kod z SMS lub aplikacji authenticator.

3

Łączy się przez garminconnect (biblioteka Python)

Biblioteka używa oficjalnego API Garmin Connect — dokładnie tak samo jak aplikacja mobilna. Sesja trwa do zamknięcia terminala.

Jak wygląda poprawne logowanie: 
Garmin email: jan@example.com
Garmin password: (hasło ukryte — wpisz i naciśnij Enter)
✓ Logged in to Garmin Connect
Błąd "SSO failure" lub "Invalid credentials": Sprawdź czy login i hasło są takie same jak na connect.garmin.com (nie konto Garmin urządzeń — to może być inne konto). Spróbuj zalogować się ręcznie na stronie — jeśli tam działa, w skrypcie też zadziała.
Masz włączone Single Sign-On przez Google / Facebook? Garmin API wymaga bezpośredniego loginu i hasła Garmin. Jeśli logujesz się przez Google, musisz ustawić oddzielne hasło Garmin: connect.garmin.com → Ustawienia konta → Zmień hasło.

📅 7. Uruchamianie: Paczka 2 — Sezon z wieloma zawodami

Skrypt season_plan.py obsługuje cały sezon. Każdy wyścig dostaje oddzielny blok treningowy z własnym prefixem. Plany nie kolidują ze sobą.

Metoda A — Plik konfiguracyjny JSON (zalecana)

Edytuj plik season_example.json w Notatniku lub innym edytorze tekstowym:

{
  "ftp":       255,
  "run_pace":  "5:20",
  "weight_kg": 80,
  "races": [
    { "name": "WARSAW",  "date": "2026-06-07", "distance": "70.3" },
    { "name": "GDYNIA",  "date": "2026-07-19", "distance": "70.3" },
    { "name": "POZNAN",  "date": "2026-08-30", "distance": "70.3" }
  ]
}

Następnie uruchom:

python season_plan.py --config season_example.json --reset

Metoda B — Tryb interaktywny

python season_plan.py --reset

Skrypt zapyta o FTP, tempo, a potem o każde zawody po kolei. Naciśnij Enter z pustą nazwą gdy skończysz dodawać zawody.

Metoda C — Podgląd bez wgrywania

python season_plan.py --config season_example.json --dry-run
Ważne zasady
Różne prefixy = brak konfliktów. WARSAW-T01, BERLIN-T01 to osobne treningi. --reset usuwa tylko treningi z danym prefixem — nie rusza pozostałych. Możesz bezpiecznie resetować jeden wyścig bez wpływu na inne.
Nakładające się daty między blokami (np. Warsaw i Gdynia): Skrypt wyświetli ostrzeżenie o datach gdzie dwa plany mają treningi tego samego dnia. Możesz je zignorować lub dostosować ręcznie w Garmin Connect. Każdy plan jest poprawny — nakładanie wynika z krótkich przerw między wyścigami.
⚡ 8. Jak skrypt konfiguruje moc i tempo w Garmin

To był najtrudniejszy aspekt techniczny — Garmin API ma nieintuicyjne kodowanie celów. Skrypt używa wartości potwierdzonych empirycznie przez analizę żądań sieciowych.

Power target — treningi rowerowe

Pole APIKrok interval/recoveryKrok warmup/cooldown
workoutTargetTypeId2 (power.zone)1 (no.target)
targetValueOnenp. 158.0 (waty)null
targetValueTwonp. 184.0 (waty)null
Dlaczego warmup/cooldown mają "no.target"? Garmin wyświetla wartości w sekcji warmup/cooldown jako prędkość (kph) gdy mają power target — błąd API odkryty empirycznie (wartość w watach × 3.6 = wyświetlane kph). Tylko kroki interval i recovery poprawnie interpretują absolutne waty.

Pace target — treningi biegowe

Pole APIWartośćPrzeliczenie
workoutTargetTypeId6pace.zone
targetValueOne/Twonp. 3.12m/s = 1000 ÷ (min×60+sek)

Przykład: tempo 5:20/km → 1000 ÷ 320 sekund = 3.125 m/s

Strefy mocy (przykład FTP=255W)

Strefa% FTPWatyZastosowanie
Z1 Recovery40–55%102–140WWarmup, cooldown, regeneracja
Z2 Endurance60–72%153–184WBaza aerobowa — główna strefa triathlonu
Z3 Tempo76–87%194–222WTempo wytrzymałościowe
Z4 Threshold88–97%224–247WProgi, interwały kluczowe
Z5 VO2max102–112%260–286WKrótkie interwały maksymalne
Race 70.379–85%201–217WTempo wyścigu 90km — indywidualnie ±3%
Jak znaleźć swoje FTP? MyWhoosh: Settings → Performance → FTP.
Zwift: Menu → Training → FTP.
Brak FTP: wykonaj 20-minutowy test maksymalnego wysiłku. FTP = 95% średniej mocy z testu.
🔧 9. Rozwiązywanie problemów

"python is not recognized" (Windows)

Python nie jest w PATH. Odinstaluj i zainstaluj ponownie zaznaczając "Add Python to PATH", lub zainstaluj ze sklepu Microsoft Store.

"execution of scripts is disabled" (Windows PowerShell)

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

"ModuleNotFoundError: No module named 'garminconnect'"

Środowisko wirtualne nie jest aktywne. Uruchom najpierw: source garmin-venv/bin/activate (Mac/Linux) lub garmin-venv\Scripts\activate (Windows). Następnie ponownie uruchom skrypt.

"SSO failure" / "Invalid credentials" przy logowaniu Garmin

Sprawdź czy login i hasło pasują do konta na connect.garmin.com. Jeśli logujesz się przez Google/Facebook — musisz najpierw ustawić osobne hasło Garmin w ustawieniach konta.

Skrypt prosi o MFA kod — co wpisać?

Wpisz 6-cyfrowy kod z aplikacji Google Authenticator, Authy lub z SMS od Garmin. Kod jest ważny 30 sekund — wpisz szybko i naciśnij Enter.

Treningi rowerowe pokazują kph zamiast watów

Stare treningi z błędnym formatem są jeszcze w Garmin. Uruchom skrypt z flagą --reset — usunie stare i wgra nowe z poprawnym formatem power target.

Duplikaty treningów w kalendarzu

Zawsze używaj flagi --reset. Skrypt usuwa stare treningi z tym samym prefixem przed wgraniem nowych.

Claude nie widzi narzędzi Strava po konfiguracji

Sprawdź plik claude_desktop_config.json pod kątem błędów JSON (walidator: jsonlint.com). Całkowicie zamknij i uruchom ponownie Claude Desktop. Sprawdź czy Node.js jest zainstalowane: npx --version.

Port 8111 zablokowany (Strava MCP nie otwiera przeglądarki)

Na Windowsie dodaj wyjątek w Zaporze Windows Defender dla Node.js. Na Linuksie: sudo ufw allow 8111.

Szybka diagnostyka — uruchom suche przejście: 
python generate_plan.py --dry-run
Jeśli skrypt pokazuje listę treningów bez błędów — instalacja jest poprawna. Błędy pojawiają się dopiero przy połączeniu z Garmin.
🔄 10. Aktualizacja planu w trakcie sezonu

Po 2–3 miesiącach treningu Twoja forma zmienia się — FTP rośnie, objętość może wymagać korekty. update_plan.py zastępuje tylko przyszłe treningi nowymi parametrami, nie ruszając już wykonanych sesji.

Kiedy aktualizować plan? Co 6–8 tygodni, po retescie FTP, po ocenie formy przez Stravę lub gdy vol-scale nie odpowiada aktualnej kondycji.

Krok 1 — Sprawdź aktualny stan planu

python update_plan.py --list       # lista wszystkich planów
python update_plan.py --prefix WARSAW  # postęp i szczegóły planu

Krok 2 — Zdecyduj co zmieniasz

--ftp 270
Nowe FTP w watach po retescie lub pomiarze MyWhoosh.
--auto-ftp
Pobiera FTP automatycznie z Garmin Connect (wymaga tokenu ~/.garmin_token).
--vol-scale 1.1
Mnożnik objętości (0.5–1.5). 1.1 = 10% więcej treningów.
--target-time 4:55:00
Nowy cel wyścigu — przeliczy tempo biegu i strefy rowerowe.
--from-strava
Automatyczna sugestia ze Stravy przed aktualizacją.
--from-date 2026-07-14
Aktualizuj od konkretnej daty (domyślnie: następny poniedziałek).
--dry-run
Podgląd bez wgrywania do Garmin.

Krok 3 — Podgląd, potem aktualizacja

# Sprawdź co się zmieni
python update_plan.py --prefix WARSAW --ftp 270 --vol-scale 1.1 --dry-run

# Wgraj do Garmin po potwierdzeniu
python update_plan.py --prefix WARSAW --ftp 270 --vol-scale 1.1

# Pełna aktualizacja przez Stravę
python update_plan.py --prefix WARSAW --from-strava
Predykcja TSB w dniu wyścigu (automatyczna) Po wygenerowaniu zaktualizowanego planu skrypt oblicza przewidywany TSB (formę) na dzień wyścigu. Cel: TSB 5–25. Jeśli wartość jest poza tym oknem, skrypt sugeruje konkretną datę --from-date do korekty taperu — wystarczy skopiować i uruchomić ponownie.
Co się dzieje z wykonanymi treningami? Treningi z datą przed cutoff (domyślnie: następny poniedziałek) zostają nienaruszone. Usuwane są tylko przyszłe zaplanowania z kalendarza Garmin i biblioteki treningów dla danego prefixu.
📊 11. Narzędzia analityczne

Cztery dodatkowe skrypty do analizy formy i planowania wyścigu. Działają offline — nie wymagają logowania do Garmin (z wyjątkiem plan_review.py).

Obciążenie treningowe — TSS/CTL/ATL/TSB

Oblicza planowane krzywi formy PMC bez łączenia z Garmin. Wymaga zapisanego stanu planu (~/.triathlon_plans/{PREFIX}.json).

python training_load.py --prefix WARSAW         # cały sezon
python training_load.py --prefix WARSAW --weeks 8 # ostatnie 8 tygodni
python training_load.py --list
MetrykaZnaczenieOkno czasowe
TSSTraining Stress Score — obciążenie jednej sesjidzienny
CTLChronic Training Load — długoterminowa forma (fitness)42 dni
ATLAcute Training Load — krótkoterminowe zmęczenie7 dni
TSBTraining Stress Balance — dyspozycja = CTL − ATL—
Interpretacja TSB w dniu wyścigu: TSB 5–25 = dobra forma ✓  |  TSB < 5 = wydłuż taper  |  TSB > 25 = skróć taper

Przegląd postępów — zaplanowane vs wykonane

Pobiera Twoje aktywności z Garmin Connect i zestawia je z planem. Wymaga aktywnego tokenu Garmin.

python plan_review.py --prefix WARSAW           # pełny przegląd
python plan_review.py --prefix WARSAW --weeks 4  # ostatnie 4 tygodnie

Wynik per sesja: ✓ wykonano (z rzeczywistym czasem/mocą/tempem) lub ✗ pominięto. Pasek ukończenia tygodnia i łączny procent planu.

Kalkulator tempa wyścigowego

Pokazuje trzy scenariusze jazdy rowerem z prognozą spowolnienia biegu i planem żywienia. Bez logowania do Garmin.

# Trzy scenariusze na podstawie FTP i wagi
python race_pacing.py --distance 70.3 --ftp 255 --weight 86

# Z docelowym czasem ukończenia
python race_pacing.py --distance 70.3 --ftp 255 --weight 86 --target-time 5:00:00

# Z ręcznym tempem biegu (świeże tempo)
python race_pacing.py --distance full --ftp 255 --weight 86 --run-pace 5:20

Eksport do kalendarza (.ics)

Generuje plik iCalendar z całym planem. Każdy trening to wydarzenie całodniowe. Wyścig zaznaczony osobno.

python export_ics.py --prefix WARSAW                    # cały plan
python export_ics.py --prefix WARSAW --future-only       # tylko przyszłe
python export_ics.py --prefix WARSAW --output plan.ics   # własna nazwa pliku
